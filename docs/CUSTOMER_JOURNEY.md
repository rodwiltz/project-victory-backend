# Big W Customer Journey

Project Victory exists to guide customers through every rental interaction while protecting Big W Events inventory in the background.

The customer should never have to understand Project Victory.

Project Victory should understand the customer.

## Journey

Reservation Confirmed
↓
Pickup Invitation Sent
↓
Customer Arrives
↓
Identity Confirmed
↓
Guided Pickup
↓
Rental Complete
↓
Event Happens
↓
Return Invitation Sent
↓
Guided Return
↓
Agreement Closed

## Core Principle

Every screen should reduce uncertainty.

Every backend service should increase confidence.

## Customer Archetype

Design for one user type:

The Customer Archetype.

If the interface is clear enough for a first-time customer, it will also work for Big W staff.

## Experience Rules

- Use Big W Events branding in customer-facing screens.
- Avoid internal terms when possible.
- Do not make the customer understand agreement logic.
- Use the agreement number internally as the system key.
- Guide the customer one step at a time.
- Show actions, not statistics.
- Remove duplicate information.
- Prefer plain language over operational language.
- Completion should be automatic when the system knows the rental step is complete.

## Pickup Experience

The customer should feel guided from the moment they arrive.

Preferred experience:

Welcome, John.

Let's get your rental.

Grab 6 Chairs.

Scan each item as you load it.

Chairs complete.

Next, grab 2 Linens.

Everything has been verified.

You're ready to go.

Have an amazing event.

## Return Experience

The return should mirror pickup.

Preferred experience:

Welcome back, John.

Let's return your rental.

Return 6 Chairs.

Scan each item as you place it back.

Chairs complete.

Next, return 2 Linens.

Everything has been returned.

Thank you for choosing Big W Events.

## System Identity vs Human Identity

System identity includes:

- Agreement number
- Cart ID
- Session ID
- Inventory item ID

Human identity includes:

- Customer name
- Pickup or return purpose
- Clear next action
- Reassuring completion message

The system may depend on agreement-level data, but the customer experience should feel personal and simple.

## Launch 1 Standard

Launch 1 succeeds when a customer can arrive at a self-service pickup location and confidently complete pickup or return without Big W staff present.

North Location cannot launch until this experience feels clear, safe, and trustworthy.
